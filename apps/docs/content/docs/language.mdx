WireScript uses S-expressions with keyword arguments:

```wire
(element-type content? properties* children*)
```

## Token Types

| Token | Pattern | Example |
|-------|---------|---------|
| String | `"..."` | `"Hello World"` |
| Number | `123`, `-45.6` | `16`, `0.5` |
| Symbol | `word-like` | `dashboard`, `user-card` |
| Keyword | `:word` | `:primary`, `:gap` |
| Hash Ref | `#word` | `#my-modal`, `#confirm` |
| Param Ref | `$word` | `$name`, `$value` |

## Property Syntax

Properties come in two forms:

```wire
(button "Submit" :primary :disabled :to checkout)
;                 ↑        ↑         ↑
;                 boolean  boolean   key-value

; Boolean: :property alone (acts as true)
; Key-value: :property value (explicit value)
```

## Document Structure

```wire
(wire
  (include "components.wire")   ; Include other files (0+)
  (meta :title "App")           ; Document metadata (optional)
  (define name (p) (text $p))   ; Component definitions (0+)
  (layout name (slot))          ; Layout definitions (0+)
  (screen id "Name" :desktop    ; Screens (1+)
    (text "Content")))
```

## Includes

Import components, layouts, and screens from other files:

```wire
(wire
  (include "components.wire")
  (include "layouts/app-layout.wire")
  (include "../common/helpers.wire")

  (screen home "Home" :desktop
    (my-component)))  ; From components.wire
```

**Rules:**
- Include paths are relative to the current file
- Supports `./`, `../` for navigation
- Multiple includes allowed
- Last definition wins for conflicts
- Library files don't need screens (components/layouts only)
- Circular includes are detected and reported as errors

**Example structure:**
```
project/
  ├── main.wire           # Entry file (requires screens)
  ├── components.wire     # Reusable components (no screens)
  └── layouts/
      └── app.wire        # Reusable layouts (no screens)
```

**Resolution behavior:**
- **VS Code**: Reads files from workspace on-demand
- **Obsidian**: Loads from vault via VFS (Virtual File System)
- **Chrome Extension**: Fetches from GitHub API via VFS
- **Node.js CLI**: Reads files from filesystem

### Meta

```wire
(meta :title "My App")
```

### Screen

```wire
(screen screen-id "Display Name" :desktop :layout layout-name
  (box :padding 24
    (text "Content")))
```

| Part | Required | Description |
|------|----------|-------------|
| `screen-id` | Yes | Unique identifier (symbol) |
| `"Display Name"` | No | Human-readable name |
| `:desktop` / `:tablet` / `:mobile` | No | Viewport size |
| `:layout layout-name` | No | Reference to a layout |

### Viewport Sizes

| Viewport | Width | Height |
|----------|-------|--------|
| `:mobile` | 375px | 667px |
| `:tablet` | 768px | 1024px |
| `:desktop` | 1280px | 800px |

## Components

Define reusable components with parameters:

```wire
(define component-name (param1 param2)
  (box :row
    (text $param1)      ; Reference with $
    (text $param2)))

; Usage - always named arguments
(component-name :param1 "Hello" :param2 "World")
```

**Rules:**
- Parameters referenced with `$param` in body
- Calls must use named arguments (`:param value`)
- Component names cannot shadow built-in elements

## Layouts

Define reusable page structures:

```wire
(layout layout-name
  (box :row :full
    (nav
      (button "Home" :ghost :to home)
      (button "Settings" :ghost :to settings))
    (slot)))  ; Placeholder for screen content

(screen home "Home" :desktop :layout layout-name
  (text "Page content"))  ; Replaces (slot)
```

**Auto-active:** Elements with `:to` matching current screen get `:active` styling automatically.

## Overlays

Overlays are defined inside screens:

```wire
(screen main "Main" :desktop
  (box (button "Open" :to #my-modal))

  (modal :id "my-modal"
    (text "Modal Content")
    (button "Close" :to :close)))
```

| Overlay | Position |
|---------|----------|
| `modal` | Centered |
| `drawer` | `:left` or `:right` side |
| `popover` | Centered (smaller) |

## Repeat

Generate repeated content:

```wire
(repeat :count 5
  (skeleton))

(repeat :count 3 :as "i"
  (text "Item ${i}"))  ; String interpolation
```
